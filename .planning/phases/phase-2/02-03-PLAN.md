<objective>
Build the `*-arr`-agnostic integration seam — the firewall that is the ONLY place *arr knowledge lives. Delivers ARR-01 (one `ArrAdapter` Protocol; Lidarr + Readarr both satisfy it; the core sees only `GapItem`), ARR-02 (uniform identity + quality profile/cutoff exposure; Readarr garbage/empty metadata degrades gracefully without crashing or gating music), and GAP-01/GAP-02 (detect monitored missing via `wanted/missing` and cutoff-unmet via `wanted/cutoff`, verified Servarr v1 routes/envelope). Phase 2 IMPLEMENTS only `get_wanted()`; the import/command/profile/queue methods are declared-and-stubbed so the seam shape is locked now without building Phase 3-5 behavior.

Purpose: Readarr's development halted in 2024 (unmaintained, fragile metadata). ARR-01 isolation + ARR-02 graceful degradation + the breaker make "books never gate music" structural, not hoped-for. Per the forward-compat directive, `GapItem` carries both the dedup identity (`arr_app`+`arr_id`) and the canonical `foreign_id` (MBID/foreign-book id) the adapter already receives.
Output: `app/adapters/base.py`, `lidarr.py`, `readarr.py`, `breaker.py`, and three test files.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Curator → Lidarr/Readarr REST | configurable base URLs; *arr responses are untrusted input crossing into the process |
| *arr response → GapItem | possibly malformed/hostile (esp. Readarr) record parsing |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07 | Information Disclosure | X-Api-Key in adapter requests | mitigate | Key injected from settings (env-only); set only as a request header; never logged (log messages reference record ids, never headers/keys) |
| T-02-08 | Denial of Service | Malformed/hostile *arr response crashes the detection loop | mitigate | Readarr `_map` returns None on non-dict/missing-id/exception; `_paged` swallows httpx errors → []; breaker returns [] when open — a bad record/fault is skipped, not fatal (ARR-02) |
| T-02-09 | Tampering / SSRF | Configurable *arr base URLs could be pointed at arbitrary hosts | mitigate | base_url comes from env wired to the expected synobridge container hosts (lidarr:8686/readarr:8787); httpx default does not follow cross-host redirects implicitly — do not enable follow_redirects to arbitrary hosts; timeout=30.0 bounds the request |
| T-02-10 | Denial of Service | A hung Readarr stalls the whole run | mitigate | timeout=30.0 per request + the breaker skips Readarr after fail_threshold consecutive failures; Lidarr iterates independently |
</threat_model>
