## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| repo → CI → Docker Hub | Secrets cross only as GitHub repo secrets; must not land in image layers |
| Docker Hub image → NAS | Generic image, host-specific config injected at runtime via .env |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-08 | Information Disclosure | Secrets baked into the published image | mitigate | No secret in Dockerfile/ENV; `.dockerignore` excludes `.env`/`.git`/`.planning`; `docker history` scan (3e) in Plan 04 |
| T-01-09 | Information Disclosure | Docker Hub credential in CI | mitigate | Use Docker Hub ACCESS TOKEN (not password) via GitHub repo secrets; `permissions: contents: read` |
| T-01-10 | Tampering | Malicious/typosquatted GH actions | mitigate | Official docker/* + actions/* orgs only (legitimacy audit); pinned majors |
| T-01-SC | Tampering | pip installs in the image | mitigate | Only fastapi + uvicorn[standard] (official PyPI, first-party); pinned versions in requirements.txt; no [ASSUMED]/[SUS] packages |
</threat_model>
