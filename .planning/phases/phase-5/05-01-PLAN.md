<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| migration DDL → SQLite | The user_version-gated migration runs on the live ~1493-row NAS DB; a malformed rebuild could lose rows |
| eligibility/backoff SQL ← Python-computed cutoffs | grace/dormant cutoffs + room are bound into SQL; if f-stringed they would be an injection surface |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering | migration_0003 table rebuild | mitigate | Explicit-column-list `INSERT ... SELECT` (not `SELECT *`) + a v0002→v0003 row-preservation test asserting count unchanged; runs inside the existing single BEGIN/COMMIT (rollback on partial failure) |
| T-05-02 | Tampering | select_eligible / record_attempt SQL | mitigate | All values `?`-bound; cutoffs computed in Python as `_now_iso()`-style strings; no f-string into SQL (repo.py security header rule, T-02-03) |
| T-05-03 | DoS (self-inflicted) | eligibility `LIMIT room` | mitigate | The `room` LIMIT is the per-cycle flood control over the ~1493-gap backlog (Pitfall 1); the SQL itself caps the result set |
| T-05-04 | Info disclosure | config tunables | accept | All new fields are non-secret integers/floats/bools; no API key/token added (D-13 defers Pushover) — nothing to leak |
| T-05-SC | Tampering | npm/pip installs | mitigate | Phase 5 installs ZERO new packages (stdlib + already-pinned fastapi/httpx); no Package Legitimacy Gate triggered this plan |
</threat_model>
