<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| scheduler workers → single sqlite conn | Bounded-concurrency workers share one connection; concurrent writes would corrupt it |
| the ~1493-gap backlog → the cycle | All past grace at launch; an unbounded dispatch would firehose Soulseek (self-DoS) |
| infra outage → attempt counter | An outage misclassified as a genuine fail would wrongly mark items permanently-unavailable |
| a cycle exception → the daemon thread | An unhandled exception would silently stop all gap-filling (anti-REL-01) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-14 | DoS (self-inflicted) | backlog flood on enable | mitigate | Per-cycle `LIMIT room` (OQ-3) + `ThreadPoolExecutor(max_workers=MAX_CONCURRENT)` hard cap + staged rollout (dry-run→cap=1→cap=3); proven by test_concurrency.py |
| T-05-15 | Tampering | two writers corrupt the connection | mitigate | LockedConn serializes every `conn.execute` under the shared writer lock; no per-worker connection (D-16); test_concurrency asserts no `database is locked` |
| T-05-16 | DoS | a cycle exception kills the daemon | mitigate | The loop body is wrapped in try/except that logs and continues (Pitfall 5); test_scheduler proves a raising cycle does not stop the thread |
| T-05-17 | Spoofing | wrong-attempt-burn on infra/usenet-active | mitigate | run_one returns infra-skip / skip-usenet-active → apply_result writes nothing (REL-02, D-02); proven by test_scheduler |
