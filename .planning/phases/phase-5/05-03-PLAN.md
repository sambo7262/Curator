## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Curator → slskd shares API | A failed shares read/rescan must not crash the cycle nor block acquisition (leech risk surfaced, not fatal) |
| reconcile → *arr verify | An *arr outage during reconcile must leave rows untouched (no burn, no double-import) |
| worker/scheduler → single sqlite conn | Every reconcile write goes through the shared writer lock (single-writer) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-09 | DoS | shares rescan retry-storm | mitigate | ensure_shares triggers exactly ONE rescan per zero-count cycle and is eventually-consistent across cycles (Pitfall 6); it never loops or re-reads in-cycle |
| T-05-10 | Tampering | double-import on restart | mitigate | verify-by-requery guard before any reset; a completed import → `imported` (execute_import never called); test asserts no double-import (Pitfall 3) |
| T-05-11 | Spoofing | wrong-attempt-burn on infra during reconcile | mitigate | INFRA_EXC → `continue` (leave as-is); a clean reset sets `pending` WITHOUT record_attempt (D-14, REL-02) |
| T-05-12 | Tampering | concurrent writers corrupt the connection | mitigate | All reconcile writes under the shared writer lock; single connection (D-16) |
| T-05-13 | Tampering | firewall regression in new core modules | mitigate | shares.py/reconcile.py speak only neutral types; auto-scanning firewall grep runs in verify |
| T-05-24 | Denial of Service (self) | `searching` orphan stranded forever (never re-eligible) | mitigate | reconcile resets `searching` (as well as downloading/importing) to `pending` so select_eligible re-picks it; test_reconcile asserts the reset (D-14, REL-02) |
