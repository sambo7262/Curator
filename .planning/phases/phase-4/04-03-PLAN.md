<objective>
Implement the explicit, *arr-agnostic import path (the entire reason this project exists, replacing Soularr's blind rescan). Music (Lidarr) raises on hard faults; books (Readarr) degrade silently.

Purpose: 04-04's loop calls these adapter methods (mapping → filtered import → verify) behind the firewall. The wire vocabulary (folder/downloadId/albumReleaseId/importMode/files[]) stays adapter-local so core/acquire.py speaks only neutral shapes. The IMPORTABILITY decision (which resources are wanted) is made HERE inside the adapter by reading *arr rejections/tracks — core never sees those keys.
Output: the three implemented ArrAdapter import methods on Lidarr (primary) + Readarr (best-effort), and their respx-backed tests.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| *arr manualimport JSON → adapter | untrusted *arr response (rejections, file mappings) crosses here; the adapter reads rejections/tracks and returns only the importable subset; core never reads its keys |
| Curator → *arr (network) | X-Api-Key auth over synobridge container-name routing |
| Readarr fault → loop | a book-side outage must not escalate into a music-path failure (ARR-02) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-08 | DoS (availability) | readarr import methods | mitigate | Whole-body swallow→safe-default on every import method (ARR-02); a Readarr 5xx/timeout can never raise into the loop |
| T-04-09 | Tampering | lidarr execute_import (wrong files imported) | mitigate | Explicit per-file files[] with resolved trackIds/albumReleaseId + Move; the adapter pre-filters to empty-rejection / non-empty-tracks resources so only importable files reach execute_import; NEVER DownloadedAlbumsScan (no blind rescan) |
| T-04-10 | Information Disclosure | X-Api-Key | mitigate | Keys in headers/params only, from settings; never logged |
| T-04-11 | Spoofing/Integrity | verify_imported false positive | mitigate | verify re-queries the *arr and returns False unless the item truly left the wanted list; Readarr verify returns False on fault (forces quarantine, never a fake success) |
| T-04-SC | Tampering | pip installs | accept | No new package (httpx/respx already pinned + approved). No install task. |
