<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| slskd peer → search/transfer JSON → SlskdClient | untrusted peer-supplied filenames + sizes + state strings cross here |
| peer filename → staging path → filesystem | a malicious filename could attempt `../` traversal out of the staging root |
| Curator → slskd (network) | reached only via gluetun published port (NAS-IP:5030), X-API-Key auth |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | Tampering / Elevation | staging.py path build | mitigate | assert_under_root resolve()+parents check before EVERY move/rmtree; reject paths escaping the staging/quarantine root (V12 path traversal) |
| T-04-05 | Tampering / DoS | staging.py purge with mis-set root | mitigate | Refuse to operate when root resolves to /, /data, or /data/media (or too few path parts) — a mis-set quarantine_root cannot wipe the library |
| T-04-06 | DoS | SlskdClient parsing untrusted JSON | mitigate | .get()-defensive on every response field (never subscript); per-call timeouts; one malformed response surfaces as a handled fault, not a crash |
| T-04-07 | Information Disclosure | SlskdClient X-API-Key | mitigate | Key lives in the headers dict only, sourced from settings; never logged, never echoed in exceptions |
| T-04-SC | Tampering | pip installs | accept | No new package (httpx/respx already pinned + approved). No install task. |
</threat_model>
