- **The matcher is a single cohesive pure module proven by corpus calibration, not by invented numbers (03-03)** — `score` + `recommend` ship together in `app/core/matching.py` (splitting mid-file = non-functional intermediate), reading only manifest identity + `candidate.parsed_*`/`audio_file_count`/`file_titles` — never uploader fields (Pitfall 5) and never year/label/catalog (Pitfall 1, anchoring rule). The **default MatchConfig calibrated the corpus correctly with NO tuning** (known_good=0.00, borderline=0.05 inside strong=0.15; wrong/incomplete/edition/garbage all >0.15; ambiguous twins decline via rec-gap), so the 03-02 config defaults stand. Calibration discipline is enforced in the test header: tune the number/weight, never the assertion (Pitfall 3). The two QUAL/fakeflac DECLINE fixtures (`fake_flac`/`below_cutoff_mp3`) are EXCLUDED from the match corpus — the matcher accepts them on identity; they are gate-layer declines proven in 03-05.

- **Phase-3 matcher/gate tunables live in the existing frozen Settings, not pydantic-settings (03-02)** — `MATCH_STRONG_THRESH`/`MATCH_REC_GAP_THRESH`/per-sub-distance weights/`FAKEFLAC_MIN_KBPS` were added in place to `app/config.py` Settings + from_env() (SP-4), each cast float/int so a non-numeric operator value fails fast with a clear ValueError at startup (T-03-12, accepted) rather than silently mis-gating. Defaults equal the MatchConfig defaults (RESEARCH 207-215) so no-env behavior is identical to the hard-coded scorer; gate.py (03-05) reads `config.settings` to build its MatchConfig. rapidfuzz pinned at **3.13.0** (current PyPI stable, verified via `pip index versions`; pre-authorized legitimate package — no human pause), with a provenance comment mirroring the Phase-2 httpx package-legitimacy checkpoint.
